# RIDI KCP Http Proxy Rest API
## 테스트 실행하기
```shell
make test
```

## 로컬 환경에서 구동하기
```shell
docker-compose up
```
cf. 필요시 docker-compose.yml의 environment를 수정해서 사용  
Sentry Logging은 environment의 `PROFILE=production`인 경우에 `SENTRY_DSN`이 지정되어 있으면 작동됩니다.

## Base URI
//api.kcp.ridi.io (미정)
